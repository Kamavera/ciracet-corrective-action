🎨 Corrective Action Form - Preview Guide

📍 You're viewing a static preview page
To see the actual working form with SharePoint data, follow the instructions below.

🚀 How to View the Real Form

Option 1: Local Workbench (No SharePoint data)

View the form UI locally without SharePoint connection:

$ nvm use 18
$ gulp trust-dev-cert
$ gulp serve --nobrowser

# Then open in browser:
https://localhost:4321/temp/workbench.html

✅ Fast preview of UI
❌ No real data (will show errors)

Option 2: SharePoint Hosted Workbench (Best for testing)

Test with real SharePoint lists:

$ nvm use 18
$ gulp trust-dev-cert
$ npm run serve --nobrowser

# Then navigate to:
https://YOUR-TENANT.sharepoint.com/_layouts/15/workbench.aspx

✅ Real SharePoint data
✅ Full functionality
✅ Code runs locally (hot reload)

Option 3: Add to SharePoint Page

Deploy to a real SharePoint page:

$ nvm use 18
$ npm run serve

# Browser opens automatically, then:
# 1. Edit page
# 2. Click + (add section)
# 3. Search "Corrective Action Form"
# 4. Add to page

✅ Production-like environment
✅ Full functionality

🎯 Recommended: Use SharePoint Hosted Workbench

For the best preview experience with real data:

  1. Run: npm run serve --nobrowser
  2. Open: https://YOUR-SHAREPOINT-SITE/_layouts/15/workbench.aspx
  3. Click the + icon
  4. Add "Corrective Action Form"
  5. Test with real data from your lists! 🚀
⚠️ Important Notes:
  • The local workbench (localhost:4321/temp/workbench.html) won't connect to SharePoint
  • For testing with real data, use the SharePoint-hosted workbench
  • Make sure your SharePoint lists exist: "Non Conformities" and "Corrective Actions"
  • The form will show errors if lists don't exist or columns are missing
